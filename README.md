# RobotSpeleo — робот-исследователь пещер

Программная модель робота, исследующего пещеры:

- движется вперёд шагами по 5 м;
- на каждом шаге сканирует поперечную плоскость;
- измеряет уровень метана и угарного газа (CO);
- строит карту рельефа;
- выводит размеры сканируемой плоскости (ширина и высота).

## Запуск

1. Скомпилируйте проект.
2. Запустите `Main.java`.

## Ожидаемый вывод

Программа выводит:

- стартовую позицию робота;
- информацию по каждому шагу (координаты, размеры плоскости, уровень опасных газов);
- итоговое количество точек на карте.
- оставшийся кабель

## Компоненты

- `Direction` — направления движения робота.
- `CableBobbin` — модель катушки с кабелем.
- `Lidar` — имитация лидара (с размерами сканируемой плоскости).
- `ScanResult` — результат сканирования (размеры плоскости).
- `Point` — точка в 3D-пространстве.
- `ReliefMap` — карта рельефа (набор точек).
- `MethaneSensor` — датчик уровня метана.
- `COSensor` — сенсор для измерения уровня угарного газа
- `Robot` — логика управления роботом.
- `Main` — точка входа, запуск исследования.
# README: Робот‑исследователь с датчиками опасности


## Краткое описание
Система моделирует робота, исследующего пространство с контролем уровней метана и угарного газа (CO). При превышении порогов — выводятся предупреждения.

## Ключевые компоненты
- **Robot** — основной управляющий класс (движение, сбор данных, логика).
- **MethaneSensor** — датчик метана (порог: 45 000 ppm).
- **COSensor** — датчик CO (порог: 10 000 ppm).
- **Lidar** — имитация лидара (сканирует плоскость).
- **ReliefMap** — карта пройденных точек.
- **CableBobbin** — контроль длины кабеля (лимит: 50 м).
- **Point** — координата в 3D‑пространстве.
- **Direction** — направления движения (FORWARD, BACKWARD, LEFT, RIGHT).
- **Main** — точка входа (запускает исследование).


## Как работает
1. Робот движется шагами по 5 м в заданном направлении.
2. На каждом шаге:
    - сканирует пространство лидаром;
    - считывает уровни метана и CO;
    - записывает позицию на карту;
    - проверяет пороги опасности;
    - выводит данные в консоль.
3. При превышении порога — печатается предупреждение.
4. При исчерпании кабеля (50 м) движение прекращается.


## Запуск
1. Скомпилируйте проект.
2. Запустите `Main`.
3. В консоли увидите:
    - позицию робота;
    - размеры сканируемой плоскости;
    - уровни газов;
    - предупреждения при опасности;
    - итоговую статистику.


## Особенности
- Пороги опасности заданы в каждом сенсоре отдельно.
- Логика легко расширяется (новые датчики, правила реагирования).
- Вывод в консоль — читаемый и структурированный.
